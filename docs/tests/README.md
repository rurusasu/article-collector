# テストガイド

## Overview

本プロジェクトのテストは Rust ユニットテストとシェル (bats) テストで構成される。

## テスト実行

```bash
# Rust ユニットテスト
cargo test --locked

# シェルテスト
bats tests/

# 全テスト（CI と同等）
cargo test --locked && bats tests/
```

## テスト構成

| 種別 | 場所 | テスト数 | フレームワーク |
|------|------|---------|---------------|
| Rust ユニットテスト | `src/*.rs` 内 `#[cfg(test)]` | 75 | Rust 標準 |
| シェルテスト | `tests/*.bats` | - | [bats](https://github.com/bats-core/bats-core) |

### Rust ユニットテスト内訳

| モジュール | テスト数 | テスト対象 |
|-----------|---------|-----------|

| `translate.rs` | 19 | プロバイダー検出、エンドポイント解決、コンテンツ抽出 |
| `youtube.rs` | 11 | JSON3 字幕抽出、XML 字幕抽出 |

## テスト記述ルール

### 必須コメント

全てのテスト関数には以下の3項目を日本語で `///` doc コメントとして記述すること:

```rust
/// 検証: 何を検証するのか（テストの目的）
/// 理由: なぜその検証が必要なのか（背景・動機）
/// リスク: この検証を行わなかった場合に何が起きるか（未検証時の影響）
#[test]
fn test_name() {
    // ...
}
```

### 例

```rust
/// 検証: script タグとその内容を完全に除去する
/// 理由: JavaScript コードが翻訳対象テキストに混入すると、翻訳品質が著しく低下する
/// リスク: alert() 等のコードが記事本文として翻訳・保存される
#[test]
fn strip_html_removes_script_tags() {
    let html = "<p>Hello</p><script>alert('xss')</script><p>World</p>";
    let result = strip_html(html).unwrap();
    assert_eq!(result, "Hello World");
}
```

### テスト設計方針

- **純粋関数をテスト対象にする**: I/O や外部 API に依存するロジックはヘルパー関数に抽出してからテストする
- **境界値を網羅する**: 空入力、長大入力、特殊文字、欠損フィールド等
- **回帰テストを追加する**: バグ修正時は、修正前に失敗するテストケースを必ず追加する
- **テスト名は振る舞いを説明する**: `test_1` ではなく `strip_html_removes_script_tags` のように

## テストカバレッジの観点

各モジュールで以下の観点をカバーする:

### fetch.rs

| カテゴリ | 観点 |
|---------|------|
| URL バリデーション | 不正スキーム拒否、正当 URL 受理 |
| ルーティング | 各サイト判定、フォールバック |
| ID 抽出 | 各プラットフォームの URL パターン |
| HTML ストリッピング | script/style 除去、クロスタグ防止、空白正規化 |

### save.rs

| カテゴリ | 観点 |
|---------|------|
| タイプ判定 | 各プラットフォーム URL、フォールバック |
| タイトルサニタイズ | 改行除去、エスケープ、長さ制限 |
| slug 生成 | 小文字化、特殊文字、ハイフン統合、長さ制限 |

### translate.rs

| カテゴリ | 観点 |
|---------|------|
| プロバイダー検出 | 各 API URL パターン、フォールバック |
| エンドポイント解決 | パス付き/なし、末尾スラッシュ、二重付与防止 |
| コンテンツ抽出 | フィールド優先順位、複数記事結合、欠損フィールド |

### youtube.rs

| カテゴリ | 観点 |
|---------|------|
| JSON3 抽出 | セグメント取得、空/改行スキップ、構造欠損耐性 |
| XML 抽出 | テキストノード取得、エンティティデコード、空白trim |
